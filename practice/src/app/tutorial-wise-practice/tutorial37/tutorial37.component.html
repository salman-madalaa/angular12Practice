    What are the template-driven Forms :
    The forms which are handle in side our template (.html) file is called template driven forms .
    Here we can handle getting and setting of values , apply the validations , show the error messages... everything we will handled inside our .html file .
    Note : When we have to show 8 to 10 fields that time template driven forms are suitable .
    Note : When we have very large forms like 20 fields a that time Reactive forms are suitable .

    step 1: After You write the form tag we have to define template variable for this form like ' #loginForm'
    step 2:Every form have its unique template variable . Syntax : #loginForm="ngForm"
    step 3:Next we have to call an event ngSubmit to submit the data.
    step 4:Next when you click on the login button you can get the empty values in the login method which is in '.ts' file
    step 5:Because you mush have bind the input box values with the " ngModel "
    Note : When we use template driven forms dont use form-control class .
    Note : When we use template driven name property must be included in input tag . Because for binding the forms with the input fields we must have name to the input field . Based on the name property the value should be bind.

    Interview Questations :
    1) When we have the submit button out side the form tag then we click on the is it submit the form ?
    A) Yes , when we use the submit button which is out we have to write click event like (click)="login(loginForm.value)"
    2) What happend when we remove the name property of the input box? A) if you remove name property of the input box then we can not access the value of the input box . In template driven forms name property must be required. based on the name property we can get the values.
