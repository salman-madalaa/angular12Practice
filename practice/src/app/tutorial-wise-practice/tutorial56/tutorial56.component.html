    {{'What is Automation Testing ?' | titlecase}} :
    --> we write some more code in our projectto test our code ,basically our code will test our application that this is working fine or not .


    {{'How does it work ? and why we need it ?' | titlecase}} :
    --> Manual testing takes too much time . because the person check every thing in Manually one by one .
    --> This automation test is very very fast.
    --> Make Test Cases of all important functions .
    --> After we will run these test cases with simple command through command line.
    --> In very quick time , we will able to run all important features of the project .

    {{'Types of Automation testing' | titlecase}} :
    --> There are 3 Types of Automation Testing . 1) Unit Testing
    2) Integration Testing
    3) end to end Testing
    Unit Testing: Test the every smallest property or functon of our project .
    Integration Testing: The unit test and its dependency . Lets say we have a functionand this function have some dependency on the data base or some APIs kind of things .
    end to end Testing: This is Testing our complete application from end to end.

    {{'importent points of automation testing' | titlecase }} :
    --> Test cases are very helpful when we have a large application . because if a tester wants to test all our application it will take lot of time .
    --> When we want release our product we want to quickly checked out our product is working or not everything working fine or not .
    --> When we just start making application we should use manual testing. Because requirements are changing every time .

    {{'automation testing angular' | titlecase }} :
    --> In angular automation test there are 3 things .
    1) Protractor -- Run Test Case on Browser .
    2) Jasmine -- Testing environment ,Run Spec file for the test case .
    3) karma -- Run test case in Multiple browser .

    {{'check config files and structure in angular' | titlecase }} :
    --> If we see the karma.conf.js it is use for run the test cases in multiple browsers . Here we can see the crome browser,firefox browser...etc
    --> here we can give the port number,browser in this file
    --> if we see the protractor.conf.js file we can see the jaspin-spec-reporter and node options ,how much interval we want to use and some config files tell the folder ,browser name ...etc
